# Landing Page Conversion Optimization

Analyze landing page screenshots and provide specific, actionable recommendations to improve readability and conversion rate.

## Core Frameworks

### The Six-Point Punch (Lead Capture Pages)

Every high-converting landing page needs:

1. **Headline with Message Match** - Does the headline directly address the ad/link that brought them here? Specificity > cleverness.
2. **Conversion-Centered Design** - Single focus, minimal distractions, clear visual path to CTA.
3. **Appropriate Media** - Images/video that reinforce the offer (real product/people > generic stock).
4. **Intro with Bullet Points** - Scannable benefits, not features. Max 3-5 bullets.
5. **Urgency Factors** - Limited time, limited quantity, or outcome-based urgency.
6. **Trust Factors** - Testimonials, logos, reviews, security badges, guarantees.

### Visual Hierarchy & F-Pattern

Users scan pages in an F-pattern: two horizontal swipes, then vertical.

**First horizontal swipe (top-left):**
- Headline must be here
- Should answer "what is this and why should I care" in <3 seconds

**Second horizontal swipe:**
- Primary CTA should be visible here
- Supporting value prop or key benefit

**Vertical swipe:**
- Social proof, trust signals
- Secondary details, FAQ
- Final CTA

### Attention Ratio

The ideal ratio is **1:1** - one page, one goal, one CTA.

Multiple competing CTAs dilute conversion. Exception: "Contact Sales" as secondary option is acceptable.

## Evaluation Checklist

### Above the Fold

| Element | Check For |
|---------|-----------|
| Headline | Clear value prop in <8 words. Uses "you" language. Specific > vague. |
| Subheadline | Expands on headline benefit, not just restates it. |
| Primary CTA | Visible without scrolling. Action-oriented text ("Get My Quote" > "Submit"). High contrast. |
| Hero image/media | Relevant to offer. Shows product/outcome, not generic stock. |

### Copy & Readability

| Element | Check For |
|---------|-----------|
| Paragraph length | Max 2-3 lines per paragraph. No walls of text. |
| Word count | <100 words converts 50% better than >500 words. |
| Benefits vs features | Lead with transformation/outcome, not specs. |
| "You" vs "We" | Customer-centric language dominates. |
| Specificity | Numbers, timeframes, concrete outcomes vs vague claims. |

### Design & Layout

| Element | Check For |
|---------|-----------|
| Visual flow | Clear hierarchy guiding eye to CTA. |
| White space | Adequate breathing room. Increases perceived value. |
| Color contrast | CTA button stands out from background. |
| Mobile readability | Text size, tap targets, single-column consideration. |
| Page load factors | Large images, excessive scripts, embedded video concerns. |

### Trust & Social Proof

| Element | Check For |
|---------|-----------|
| Testimonials | Specific, named, with results. Photos add credibility. |
| Logos | Recognizable clients/partners/press mentions. |
| Reviews/ratings | Third-party validation (G2, Trustpilot, etc.). |
| Guarantees | Money-back, satisfaction, security badges. |
| Numbers | Customer count, years in business, results metrics. |

### CTA Optimization

| Element | Check For |
|---------|-----------|
| Button text | Action verb + benefit ("Start Free Trial" > "Submit"). |
| Button size | Large enough to be obvious, not overwhelming. |
| Placement | Above fold AND at end of page. |
| Surrounding copy | Reduces friction (no credit card, cancel anytime, etc.). |

### Form Optimization (if applicable)

| Element | Check For |
|---------|-----------|
| Field count | Fewer = higher conversion. Only ask what's essential. |
| Field labels | Clear, inside or above field. |
| Error handling | Inline validation, clear error messages. |
| Progress indication | For multi-step forms. |

## Output Format

For each recommendation, provide:

```
### [Priority: High/Medium/Quick Win] - [Section/Element]

**Issue:** What's wrong and why it hurts conversion.

**Recommendation:** Specific change to make.

**Implementation:**
- Exact copy suggestion (in quotes) OR
- CSS/design direction OR
- Structural change needed

**Why it matters:** The CRO principle behind this change.
```

Group recommendations by priority:
1. **High Impact** - Likely to significantly improve conversion
2. **Medium Impact** - Will improve UX and conversion
3. **Quick Wins** - Easy fixes with incremental benefit

## Key Statistics to Reference

- 94% of first impressions are design-related
- Users expect pages to load in <3 seconds (83%)
- Pages with <100 words convert 50% higher than >500 words
- 97% of consumers use reviews/testimonials in purchase decisions
- "Learn More" CTAs get 26% click rate with brand-aware visitors
- 10 seconds to form first impression
- 40-60% bounce rate is normal for service businesses

## Common Issues to Flag

### Immediate Red Flags
- No clear CTA above the fold
- Headline doesn't explain what the product/service is
- Generic stock imagery
- Multiple competing CTAs
- Walls of unbroken text
- No social proof visible
- Form asks for too much information

### Subtle Issues
- "We" language dominates over "you"
- Features listed without benefits
- CTA button doesn't contrast with background
- Trust signals buried below fold
- No urgency or reason to act now
- Mobile experience likely broken (multi-column, small text)
